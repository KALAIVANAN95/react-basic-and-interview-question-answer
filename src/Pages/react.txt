3.What is the meaning of Virtual DOM.?
i)A virtual DOM is a simple JavaScript object that is the exact copy of the corresponding real DOM.
or
A virtual DOM is a light weight js object.

Virtual DOM is a lightweight copy of the Real DOM.
When state or props change, React first updates the Virtual DOM.
Then React compares the new Virtual DOM with the previous Virtual DOM using a process called Diffing.
Finally, React updates only the changed parts in the Real DOM instead of re-rendering the entire DOM.

or
Virtual DOM is a virtual representation of the Real DOM. React uses it to identify the minimum changes required and update only those changes in the Real DOM, improving performance.

Reconciliation is the process React uses to compare the old Virtual DOM with the new Virtual DOM.
After comparison, React identifies the changed elements and updates only those specific parts in the Real DOM instead of updating the entire DOM.
or
Reconciliation is the process where React compares the previous Virtual DOM with the new Virtual DOM.
After finding the differences (Diffing), React updates only the changed parts in the Real DOM.

State change or props change
↓
New Virtual DOM created
↓
Old Virtual DOM vs New Virtual DOM
↓
Diffing
↓
Reconciliation
↓
Update only changed nodes in Real DOM

30.Hooks?
Hooks are special functions in React that allow functional components to use React features.
In functional components, Hooks are used to:
i) manage state
ii) handle lifecycle-like behavior
iii) access the DOM
Hooks can also manage stateful logic and side effects.

useState:
useState hook is used to create and update state in functional components.

Diff useState and useRef
useState changes cause re-renders.
useRef changes do not cause re-renders.

useEffect: 
useEffect is a React Hook used to manage side effects in functional components, such as API calls, subscriptions, timers, and DOM updates.
It works like:
componentDidMount (runs after first render)
componentDidUpdate (runs after updates)
and can also act like componentWillUnmount (cleanup)

useContext: 
useContext is used to access global/shared state in React components without prop drilling.

useReducer:
useReducer is a React Hook used for managing complex state logic.
It is an alternative to useState and is useful when state transitions depend on multiple actions or when the state contains multiple related values.

It works using:
1. State
2. Action
3. Reducer Function
4. Dispatch
useMemo
→ Returns Memoized Value
useCallback
→ Returns Memoized Function
React.memo
→ Memoizes Component

useCallback: 
useCallback is a React Hook that returns a memoized function.
It is used to prevent unnecessary recreation of functions on every render.
It returns a memorized version of a callback to help a child component not re-render unnecessarily.

useMemo:
useMemo is a React Hook that returns a memoized value.
It is used for performance optimization by avoiding expensive recalculations on every render.
ii)The useMemo Hook only runs when one of its dependencies update.

useRef:
useRef is used to access DOM elements and persist mutable values. OR
It returns a ref object with a .current property. The ref object is mutable. It is mainly used to access a child component imperatively.
useLayoutEffect: 
It fires at the end of all DOM mutations. It's best to use useEffect as much as possible over this one as the useLayoutEffect fires synchronously.

useDebugValue:
useDebugValue is used inside custom hooks to display a label in React DevTools, making it easier to debug and inspect hook values

38.What is lifting state up?
Lifting state up means moving shared state to the nearest common parent. OR
When multiple child components need shared data, state is moved to the common parent component and passed using props.
Example Scenario
Search input component
Search results component
Both need the same search text, so the state is maintained in the parent component and shared via props.
