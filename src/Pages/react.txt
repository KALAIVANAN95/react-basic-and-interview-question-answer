30.Hooks?
React Hooks are simple JavaScript functions that we can use to isolate the reusable part from a functional component.
Hooks can be stateful and can manage side-effects.
useState:
useState hook is used to create and update state in functional components.
useEffect: 
To manage side-effects like API calls, subscriptions, timers, mutations, and more.
componentDidmount and componentdidupdate - it will be done this work
useContext: 
To return the current value for a context.
useReducer:
useReducer manages complex state logic using reducers and dispatched actions. OR
A useState alternative to help with complex state management.
useCallback: 
i)useCallback memoizes functions to prevent unnecessary recreations. OR
i)It returns a memorized version of a callback to help a child component not re-render unnecessarily.
useMemo:
i)useMemo memoizes expensive calculations to improve performance. OR
i)It returns a memoized value that helps in performance optimizations.
ii)The useMemo Hook only runs when one of its dependencies update.
useRef:
useRef is used to access DOM elements and persist mutable values. OR
It returns a ref object with a .current property. The ref object is mutable. It is mainly used to access a child component imperatively.
useLayoutEffect: 
It fires at the end of all DOM mutations. It's best to use useEffect as much as possible over this one as the useLayoutEffect fires synchronously.
useDebugValue:
Helps to display a label in React DevTools for custom hooks.

33.Why dependency array important?
Dependency array controls when the effect should execute.
[] -> runs once
[value] -> runs when value changes
No dependency -> runs on every render
